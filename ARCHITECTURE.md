# 🍷 Винная Викторина — Архитектура проекта

## Обзор

Мультиплеерная игра «Угадай вино»: хост загадывает параметры вина (цвет, сладость, сорт, страна, год и т.д.) и при желании загружает фото бутылки; участники по очереди выбирают свои варианты по каждой характеристике. Побеждает тот, чьи догадки ближе всего к загаданному. Основной интерфейс — браузер, с поддержкой планшетов и смартфонов (Android, iOS) через PWA.

## База данных: стратегия развёртывания

### Принцип: локальная разработка → облачный продакшн

Вся работа с БД идёт через **Prisma ORM**. Подключение определяется единственной переменной окружения `DATABASE_URL` в файле `.env`. Для переноса на любой облачный сервис достаточно **заменить строку подключения** — код приложения не меняется.

### Окружения

| Окружение | Подключение | Описание |
|-----------|-------------|----------|
| **Локальная разработка** | `postgresql://user:password@localhost:5432/winequiz` | PostgreSQL установлен на машине разработчика |
| **Продакшн (облако)** | `postgresql://user:password@host:5432/winequiz?sslmode=require` | Любой облачный провайдер |

### Рекомендуемые облачные сервисы для переноса

| Сервис | Бесплатный план | Особенности |
|--------|----------------|-------------|
| **Supabase** | 500 МБ, 2 проекта | Встроенный Auth, Realtime, REST API |
| **Neon** | 512 МБ, branching | Serverless, автомасштабирование, мгновенный старт |
| **Railway** | $5 кредит/мес | Простой деплой, авто-бэкапы |
| **Vercel Postgres** | 256 МБ | Нативная интеграция с Next.js / Vercel |
| **AWS RDS** | 12 мес. free tier | Максимальная гибкость, масштабируемость |

### Файл `.env`

```env
# === Локальная разработка ===
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/winequiz"

# === Продакшн (пример для Supabase) ===
# DATABASE_URL="postgresql://postgres:[PASSWORD]@db.[PROJECT].supabase.co:5432/postgres?sslmode=require"

# === Продакшн (пример для Neon) ===
# DATABASE_URL="postgresql://[USER]:[PASSWORD]@[HOST].neon.tech/winequiz?sslmode=require"

# === NextAuth и приложение ===
# NEXTAUTH_URL, NEXTAUTH_SECRET, NEXT_PUBLIC_APP_URL, NEXT_PUBLIC_APP_NAME

# === Socket.io (клиент подключается к отдельному серверу) ===
# NEXT_PUBLIC_SOCKET_URL="http://localhost:3001"
```

### Миграции (Prisma)

Prisma обеспечивает **полную переносимость схемы** между окружениями:

```bash
# Разработка — создание и применение миграций
npx prisma migrate dev --name init

# Продакшн — применение миграций на облачной БД
npx prisma migrate deploy

# Заполнение начальными данными (достижения и т.д.)
npx prisma db seed
```

### Важные правила

1. **`.env` НЕ коммитится в git** — добавлен в `.gitignore`
2. **`.env.example`** — шаблон с описанием переменных (без секретов), коммитится
3. **Миграции коммитятся** — папка `prisma/migrations/` всегда в git
4. **Seed-данные коммитятся** — `prisma/seed.ts` с достижениями и прочими начальными данными в git

## Ключевые параметры

- **Максимум игроков в сессии:** 99
- **Подключение игроков:** по QR-коду или по ссылке (URL вида `https://domain/join/WN-XXXXXX`)
- **Тема интерфейса:** светлая / тёмная (next-themes, сохраняется в cookie)
- **Игровой процесс:** хост создаёт раунды с параметрами вина и опционально фото; игроки по шагам выбирают цвет, сладость, состав, сорта, страну, год, крепость, выдержку в бочке; очки начисляются за совпадения
- **Мультиплеер:** реальное время через Socket.io (отдельный сервер, по умолчанию порт 3001)
- **Аутентификация:** по телефону (Credentials), гостевой режим для быстрого входа в игру
- **Геймификация:** XP, уровни, достижения, рейтинг; профиль может быть публичным или скрытым

## Система геймификации

### XP и уровни

- **XP:** начисляется по результатам раундов и игр (совпадения с загаданным вином, место в рейтинге)
- **Уровень:** привязан к накопленному XP (формула в коде/БД)
- **Максимальный уровень:** 50

### Звания (привязаны к уровням)

| Уровни | Звание |
|--------|--------|
| 1–5 | Новичок |
| 6–15 | Любитель |
| 16–25 | Знаток |
| 26–35 | Сомелье |
| 36–45 | Энолог |
| 46–50 | Мастер вина |

### Достижения

- Шаблоны хранятся в БД (Achievement), условия задаются полем `condition` (JSON)
- Примеры типов: первая игра, серия побед, количество правильных совпадений и т.д.
- Награда: XP (xpReward); выдача через UserAchievement

### Рейтинг и профиль

- Профиль игрока: уровень, XP, достижения, история игр
- Публичность профиля: флаг `isProfilePublic` (скрытый/публичный)
- Рейтинг: по общему XP или по результатам игр (топ в профиле и на отдельной странице при наличии)

## Подключение игроков (QR-код / ссылка)

1. Хост создаёт игру через API → в БД создаётся GameSession с уникальным кодом (напр. `WN-482917`)
2. На экране хоста отображается QR-код и/или ссылка: `https://domain/join/WN-482917`
3. Игрок сканирует QR или переходит по ссылке → открывается страница `/join/[code]`
4. Игрок входит (логин/регистрация по телефону) или как гость → присоединяется к игре через API и Socket.io
5. В лобби хоста отображается новый участник; хост управляет раундами и запуском игры

**Библиотека:** `qrcode.react` для отображения QR на стороне хоста.

## Технические решения для 99 игроков

| Аспект | Решение |
|--------|---------|
| Socket.io | Отдельный процесс (`npm run socket`), комнаты по коду игры — бродкаст в комнату |
| Состояние игры | Активные комнаты в памяти (GameRoom); при перезапуске сокет-сервера хост переподключается и восстанавливает состояние из БД |
| Раунды | Хост заполняет параметры вина (Round), игроки по шагам отправляют догадки (PlayerGuess); сервер считает очки за совпадения |
| Переподключение | Socket.io авто-переподключение; хост при реконнекте получает актуальный статус из БД |
| Лобби | Список игроков в комнате, счётчик подключённых; при большом числе — виртуализация при необходимости |
| Результаты | Итоги раунда и игры через события round_closed / game_finished; отдельная страница scoreboard для трансляции |
